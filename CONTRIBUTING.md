# Contributing to SARA

Gracias por querer contribuir a SARA. A continuación encontrarás un flujo y normas mínimas para colaborar de forma efectiva.

1. Haz fork del repositorio y crea una branch con un nombre descriptivo:
   git checkout -b feature/mi-mejora

2. Escribe commits pequeños y atómicos con mensajes claros y prefijos tipo Conventional Commits (opcional pero recomendado):
   - feat: nueva funcionalidad
   - fix: corrección de bug
   - docs: cambios en documentación

3. Añade pruebas cuando añadas funcionalidad y asegúrate de que todo compila.

4. Formatea el código con las convenciones del proyecto y documenta los cambios.

5. Abre un Pull Request:
   - Describe brevemente el problema y la solución.
   - Añade capturas o pasos para probar los cambios.
   - Marca si el PR rompe compatibilidad (breaking change).

6. Revisión y merge:
   - Responde a las revisiones de forma amable y rápida.
   - Si el PR necesita cambios, actualiza la branch y los commits.
